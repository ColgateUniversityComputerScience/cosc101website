Introduction

The goal of this homework is to create a well-designed game similar to Wheel of Fortune, where the computer picks a secret word from a list, shuffles it, and the user guesses what the word is letter by letter; however, the vowels are visible to the user this time.

Please write your code in the template file hw7_wheel.py included with this homework. Note that the file includes a function (already written!) for drawing the scoreboard, a main function for your program, a few helper functions (hide_consonants, shuffle_word, and is_vowel) and the start of a function called one_game.

Required functionality

Here are the basic guidelines for how your program should behave:

  • The “scoreboard” that you draw will be in low-tech ASCII (just letter, numbers and/or symbols). (We won’t be using Turtle or another graphics package.) A function called scoreboard is provided in the hw7_wheel.py template file. Given an integer misses representing the number of missed guesses, it prints the scoreboard and misses number of X’s.

    There should be exactly seven missed guesses allowed before declaring that a player has lost.

  • The first line in function one_game hides the consonants in the secret word; the output is stored in the variable state, which corresponds to the current state of guessing, i.e., the game starts with the vowels visible; however, the letters are not in the correct order.

  • Before you ask for a new letter to guess, you should display the letters that have been previously guessed, and the current “state” of guessing (variable state). For the current guess state, you can display the actual letters that have been correctly guessed, and underscores for letters that are yet to be guessed correctly. For example, if the word is “SUNNY”, the secret word is “NYNSU” and the player has correctly guessed the “N”, you should print something like:

    N _ N _ U

    Note that the blanks are formed with underscore characters. Note also that the output is rather nicely formatted.

  • Keeping track of the guess state, or information about the word a user is trying to guess and which letters have been correctly guessed is a key challenge for this homework. You can do this with the variable state which can be updated with the following steps:

    • When a valid guess is obtained, you can iterate through the indexes of the secret word and employ an accumulator pattern to build the new “guess state”. For each index, if the letter in the secret word at that index is the same as the guess, concatenate the letter that was guessed correctly, otherwise concatenate a _.

    • Note that when you print out the guess state you will want to print a space between each letter so that it is clear how many letters are to be guessed. For example, if the secret word is PUMPKIN and the player has correctly P, the guess state should be 'P__P___'. Instead of printing 'P__P___', though, you should print 'P _ _ P _ _ _' (spaces between each letter/underscore). A recipe for accomplishing this is (assuming a string variable named game_state holds the game state):

      print(' '.join(list(game_state)))

  • A main function is also started in the hw7_wheel.py template. The main function contains a list of words; the random.choice function is used to randomly select one of the words, which is then mixed to start the game. The list of words can be modified.

  • Your program should just play one game of wheel of fortune. When a game ends because of a win or a loss by the player, the program should end.

  • When you ask for the next guess from a player, you should only accept a single letter as a valid guess; all other inputs should be treated as invalid. This behavior must be implemented in the function is_valid_guess, it contains doctests that illustrate its functionality. The program should not crash, an appropriate error message should be shown, and the game should continue. Any invalid guesses should not count against a player.

  • When you obtain a new valid guess (one letter), it is strongly suggested that you convert any inputs to upper-case for handling the rest of the game logic. Handling all strings in upper case will simplify logic in your code.

  • Any duplicate guesses should not count against a player. For example, if a player has already guessed “A” and they guess it again, you should print a message like “You’ve already guessed A”, but otherwise do nothing.

  • Once that you’re sure about how the game is played and you understand the basic requirements listed above, you’ll need to think carefully about how to design and implement the game in Python. Ensure that all your functions are SOFA: that they are short, do one thing, take few arguments, and are pitched at a consistent level of abstraction. Moreover, make sure that your functions contain well-formed docstrings and that you include comments next to portions of code that need some further explanation.

Suggested program structure

The design of the program to play Wheel of Fortune is up to you, but here is a suggested structure. This is a suggestion; if you deviate from it, that’s up to you, but for all functions you write you should try to make sure that:

  • any function you write does one thing (i.e., one specific task);
  • any function function you write just takes parameters and returns a return value (or return values), but does not modify any global state or global variables;
  • any function you write is fairly short; it is possible to write all the code to play a game in a single function but that’s a bad design.

If you go with the suggested structure below, a further strong suggestion is to write a little bit of code at a time (one function at most, but maybe just a couple lines in a function) and test as you go. Writing most all the code you think you need and testing afterward is a strategy that will often lead to frustration. Do a little bit at a time.

1. A function to get the next guess.

A key task in this game is to ask the user for their next guess (a single letter). This function should repeatedly ask the user for their guess until they type a single letter.

Some specific suggestions about this function:

  • Use a while loop for repeatedly asking the user for new input until valid (one letter) input is given.
  • Don’t worry about whether the user has guessed a particular letter before or not in this function.
  • Return the (valid) letter guessed as an upper case letter. That way, you can just deal with upper-case letters and strings elsewhere in the program which will simplify things a lot.

An example function header might be:

def get_guess():
  """Get the next player guess."""

2. A function to print the current “game state”.

There are three aspects to the game state:

  • the number of “misses” (letter guesses that aren’t in the secret word) a user has made — you need this number for calling the function to print the scoreboard,
  • all the previous guesses made by the user, whether or not those guesses were valid or not,
  • some representation of which letters have been correctly guessed and which have not yet been guessed (the “guess state”) — see notes in the section above about how you might consider storing the guess state.

This function would print this information nicely to the screen (see the game traces linked below for examples of how your output might look). This function wouldn’t need to return anything.

An example function header might be:

def print_game_state(misses, previous_guesses, guess_state):
  """Print current state of the game."""

3. A function to take a valid and correct guess, and update the guessing state.

For this function, you would accept the secret word, the guessed letter, and the current guess state, then update the guess state. The doctests can help understand the functionality of this function.

An example function header might be:

def update_game_state(guess, state, secret):
  """str, str, str -> str
  Update game state based on most recent guess.
  >>> update_state('G', 'O_O_', 'OGOD')

  >>> update_state('X', 'O_O_', 'OGOD')

  >>> update_state('D', 'O_O_', 'OGOD')

4. A function to tie together the other functions and coordinate playing one game of wheel of fortune.

If you have the above three functions written, a function called one_game could tie everything together by doing the following:

  • Initialize the guess state, including a variable to hold the number of misses, a string to hold previous (valid) guesses, and a string to hold which letters have been correctly guessed.

  • Have a while loop that will execute as long as the game is in progress; remember that the game ends when a player either has 7 misses or correctly guesses the word.

    • Next, you could ask for the next guess.
    • You could then handle the new guess by updating the guess state if it had not been previously guessed and was a letter in the secret word, otherwise you could handle the guess in another appropriate way.

Challenge problems

There are two options for challenge problems for this homework, as described below.

Challenge problem option 1: Evil Wheel of Fortune

If you complete the Wheel of Fortune game as described, you can try to complete this optional challenge problem. Modify the game to have an “evil” option. Prompt the user to play evil or regular Wheel of Fortune. If the user chooses to play evil Wheel of Fortune, then the computer does something really sneaky: it avoids committing to a secret word for as long as possible. (You’ll need to create a larger set of secret words in order to make evil Wheel of Fortune truly evil.)

Here’s how it might work. The computer has a list of words to choose from. Each time the user guesses a letter, the computer can tell the user that the letter is not in the word, provided that the computer still has some words left in its list that don’t have that letter. If eventually every word in the evil computer player’s list has the guessed letter, then the computer can finally commit to a word (by choosing one at random). To get full credit, your evil player can cheat in this way but must ultimately still play a game that appears to be fair. For instance, at the end of the game, win or lose, the computer player must expose its secret word. So, if the player guesses ‘a’ then ‘e’ then ‘o’ then ‘u’ then ‘i’ then ‘y’ and the computer says that none of those letters are in the word, then the computer player is in trouble unless it can come up with a word that has no vowels in it!

Challenge problem option 2: Fancy scoreboard

You’ll notice that the scoreboard printed by the scoreboard function is pretty … bland. For this challenge problem, you could design a new, fancier scoreboard to be printed as part of game play. For example: you could use Emoji for some parts of the scoreboard, or you could print some features in color. The design is entirely up to you.

For printing Emoji, you will either need to copy/paste particular emoji into your program, or (better) you can use the Unicode character code for a particular symbol or emoji. To use a character code in Python, you’ll need to first look up a symbol and its code equivalent; you can do this using code charts available from unicode.org. There is a page with all code charts (http://www.unicode.org/charts/), and specific pages linked from that index show Emoticons (https://www.unicode.org/charts/PDF/U1F600.pdf), miscellaneous symbols (https://www.unicode.org/charts/PDF/U2600.pdf) and more. For example, the first Emoticon face on the Emoticons table (https://www.unicode.org/charts/PDF/U1F600.pdf) lists a character code of 1F600. To use that code in Python, you need to prefix the code with '\U000', like:

print("\U0001F600")

Any Unicode character code can be printed in that way. (Note that the number of characters after '\U' must be 8, so if the character code is only 4 characters, you need to prefix it with '\U0000'.)

For printing text in color, you can refer to the following web page for examples of using “ANSI Escape Codes”: https://www.lihaoyi.com/post/BuildyourownCommandLinewithANSIescapecodes.html. The first part of that page gives some guidance on emitting specific special codes with print to cause text to be shown in different colors.
